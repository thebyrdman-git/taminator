# Taminator v2.1.5 — Release notes

**Release date:** TBD  
**Previous release:** v2.1.4

This release fixes version display in the installed app and improves build/release workflow.

---

## Fixes

### Version number in the UI
- **Issue:** The app showed "Version —" instead of the actual version (e.g. 2.1.5) after installation from the DMG.
- **Fix:** The version is now passed from Electron to the web server via the `TAMINATOR_APP_VERSION` environment variable (from `app.getVersion()`), so the installed app always shows the correct version in the hero without depending on a VERSION file in the bundle.
- **Fallbacks:** The server still reads a `VERSION` file when the env is not set (e.g. when running `tam-rfe serve` outside the app). The initial HTML is injected with the version so it appears on first load; the UI also retries `/api/version` if the first request fails.

### Build and release
- **VERSION in package:** The `VERSION` file is included in the app bundle (`extraResources`) so the server can use it when the env is not set.
- **GitHub Actions:** The 2.1.5 workflow records build ref/SHA, validates version and icon, lists artifacts before upload, and optionally creates a GitHub Release when "Create release" is enabled (tag must exist first).
