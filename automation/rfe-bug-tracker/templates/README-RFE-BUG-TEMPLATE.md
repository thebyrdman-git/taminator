# Standardized RFE/Bug Tracker Template

## Overview
This is the standardized template for generating RFE/Bug tracker reports organized by JIRA component categories. This template is distinct from the Active Case Report template and focuses specifically on RFE and Bug tracking.

## Key Characteristics

### **RFE/Bug Template vs Active Case Report**
- **RFE/Bug Template**: JIRA-focused, component-organized, no priority column
- **Active Case Report**: Case management focused, status-organized, includes priority

### **Template Structure**
1. **Executive Summary** - JIRA-focused metrics
2. **RFE Section** - All Request for Enhancements organized by JIRA components
3. **Bug Section** - All Bug reports organized by JIRA components
4. **Component Distribution Analysis** - Separate analysis for RFEs and Bugs
5. **Strategic Priorities** - Component-based prioritization

## Usage Instructions

1. **Replace placeholder data** with actual case information
2. **Update case counts** and percentages in distribution analysis
3. **Customize business impact** statements for each component
4. **Adjust strategic priorities** based on customer needs
5. **Update generation date** and metadata

## Template Features

- **JIRA Status Only** - No internal case management statuses
- **Component Organization** - Logical grouping by JIRA components
- **Separate RFE/Bug Sections** - Clear distinction between enhancement requests and bugs
- **Business Impact** - Value statements for each component category
- **Strategic Analysis** - Component distribution and prioritization
- **Professional Formatting** - Customer-ready presentation
