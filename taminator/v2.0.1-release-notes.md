# Taminator v2.0.1 — Bug fixes, credentials, Vault, and cleanup

**Release date:** TBD  
**Previous release:** v2.0.0

This release adds credential encoding, optional HashiCorp Vault integration, JIRA Cloud support, and a pass of audit fixes (generic defaults, no hardcoded contacts, cleaner build/CI).

---

## JIRA Cloud

- You can use **JIRA Cloud** (e.g. `https://your-tenant.atlassian.net`) in addition to Red Hat JIRA (issues.redhat.com). In **Settings → Token configuration → JIRA instance**, set **JIRA base URL**; when the URL is a Cloud host (`*.atlassian.net`), enter your **Email** and **API token** (from [id.atlassian.com/manage-profile/security/api-tokens](https://id.atlassian.com/manage-profile/security/api-tokens)). Check, Update, and report links use the configured instance.
- **In-app guidance** matches the configured JIRA: when the base URL is Cloud, “How to obtain” instructions and `tam-rfe config --add-token` point to the Atlassian API token page; when Red Hat, they point to issues.redhat.com Personal Access Tokens. See **User Guide → JIRA Cloud** and [docs/JIRA_CLOUD_INTEGRATION.md](docs/JIRA_CLOUD_INTEGRATION.md).

## Credentials and secret storage

### Encoded token storage (no plaintext on disk)
- JIRA, Portal, and Hydra tokens saved from Settings are stored in `~/.config/taminator/ui_tokens.json` as a **base64-encoded payload**, not as plaintext. The file uses the same permissions (mode 0600) as before. Existing plaintext files are still read; the next save upgrades them to the encoded format. See **User Guide → Credentials & Vault** and [SECURITY-PROTECTION-LAYERS.md](taminator/SECURITY-PROTECTION-LAYERS.md).

### Optional: HashiCorp Vault
- When **VAULT_ADDR** and **VAULT_TOKEN** are set, Taminator can use HashiCorp Vault as a secret store. Tokens are read from Vault when available (with fallback to the local encoded file), and saving a token from Settings syncs it to Vault as well. Optional **VAULT_NAMESPACE**, **VAULT_CACERT**, and **VAULT_SKIP_VERIFY** are supported. The **tam-vault** CLI (`status`, `migrate`, `list`, `get`/`set`/`delete`, `test`) and the in-app Vault tab are unchanged. See **[docs/VAULT_INTEGRATION.md](docs/VAULT_INTEGRATION.md)** for setup and security notes.

## Bug fixes and behavior changes

- **Token saving:** Saving from Settings writes to the local encoded file and, when Vault is configured, to Vault (web server and desktop app).
- **Server startup:** Improved PATH and Python resolution for the embedded server (e.g. when running as a different macOS user). Error dialog now shows server stderr and a Terminal workaround.
- **Vault client:** No default server address; **VAULT_ADDR** is required. SSL verification respects **VAULT_CACERT** and **VAULT_SKIP_VERIFY** (see VAULT_INTEGRATION.md).
- **RFE verification tests:** PAI-related test paths use the **PAI_SRC** env var instead of a hardcoded path.

---

## Audit cleanup (generic defaults, no hardcoded contacts)

- **Default TAM / contact:** Report and onboarding defaults no longer use a specific name or email. Default TAM name is blank or “TAM”; default contact is blank. Users set their own in Settings or onboarding.
- **Help and feedback:** In-app “Need help” and error guidance point to **GitLab issues** (https://gitlab.cee.redhat.com/jbyrd/taminator/-/issues) instead of a personal email. Feedback in Settings also points to GitLab issues.
- **Onboarding and templates:** Onboarding wizard default TAM name is “TAM”. Report and call-note templates use “Red Hat TAM” or “TAM” instead of a specific name. Contact lines in templates point to the GitLab project.
- **auth_box:** Kerberos check no longer has an unresolved TODO; help text uses the GitLab issues link.
- **Build / CI:** ESLint warnings in the GUI are resolved. `.gitignore` includes `taminator-copy-for-test/` and `gui/dist-arm64-build/` so local test copies and alternate build output are not committed.

## Platforms (unchanged)

- **Linux:** AppImage (x86_64 and ARM64). On Fedora you may need e.g. `sudo dnf install zlib` if you see `libz.so` errors.
- **macOS:** DMG (Intel and Apple Silicon). Unsigned; first open: right-click → Open if Gatekeeper blocks.
- **Windows:** NSIS installer (x64).

## Upgrade from v2.0.0

Replace the app with the v2.0.1 build for your platform. Existing tokens in `~/.config/taminator/ui_tokens.json` (plain or encoded) continue to work; the next save will store them in the new encoded format. No config migration required. To use Vault, set **VAULT_ADDR** and **VAULT_TOKEN** before starting the app or `tam-rfe`; see [docs/VAULT_INTEGRATION.md](docs/VAULT_INTEGRATION.md).
