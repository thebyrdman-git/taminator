# 🧪 Taminator v1.9.5 - User Testing Guide

**Version:** v1.9.5  
**Release Date:** October 23, 2025  
**Purpose:** Guide for testing the new release

---

## 🎯 What Changed in v1.9.5?

This release **removes fake features** and **adds honesty** to the UI. We're not adding new functionality - we're being honest about what works vs what doesn't.

---

## ✅ **What to Test**

### Test 1: Launch & Navigation (2 min)
**Goal:** Verify app launches and navigation works

**Steps:**
1. Launch Taminator v1.9.5
2. Click through each tab:
   - Home (Dashboard)
   - Verify (Check)
   - Update
   - Post
   - Onboard
   - Vault
   - Settings
   - Report Issue

**Expected:**
- ✅ App launches without errors
- ✅ All tabs load successfully
- ✅ No crashes or blank screens

**Known Issue from v1.9.3:** Icon loading bug - FIXED in v1.9.4, should work now

---

### Test 2: Dashboard Warnings (1 min)
**Goal:** Verify demo data is clearly marked

**Steps:**
1. Go to Home (Dashboard) tab
2. Look for warning banner at top
3. Check card titles

**Expected:**
- ✅ Yellow warning banner says "Dashboard shows placeholder data"
- ✅ Customer card labeled "(Demo Data)"
- ✅ Recent Activity labeled "(Demo Data)"
- ✅ Auth Status labeled "(Live)"

**Why this matters:** Previous version showed fake data without any warning - users thought it was real!

---

### Test 3: Update Tab - No More Fake Checkboxes (2 min)
**Goal:** Verify misleading checkboxes are removed

**Steps:**
1. Go to Update tab
2. Look for checkboxes

**Expected:**
- ✅ NO checkboxes for "Preserve notes"
- ✅ NO checkboxes for "Generate changelog"
- ✅ Blue info box explains what actually happens
- ✅ Text says "automatically preserved"

**Why this matters:** Previous version had checkboxes that collected values but were completely ignored by the backend!

---

### Test 4: Report Issue Tab - Honest Contact Methods (2 min)
**Goal:** Verify fake GitHub submission is removed

**Steps:**
1. Go to Report Issue tab
2. Look for form

**Expected:**
- ✅ Yellow warning: "Direct GitHub submission is not yet implemented"
- ✅ Shows real contact methods:
  - Email: jbyrd@redhat.com
  - GitLab Issues link
  - Slack: #tam-automation
- ✅ NO bug report form with "Submit" button

**Why this matters:** Previous version had a form that returned FAKE success - users thought they reported bugs but didn't!

---

### Test 5: Tab Descriptions (3 min)
**Goal:** Verify all tabs explain what they do

**Steps:**
1. Click each tab and look for blue info boxes

**Expected:**
- ✅ **Verify tab:** "What this does: Compares your saved RFE/Bug tracker..."
- ✅ **Update tab:** "What this does: Fetches current status from JIRA..."
- ✅ **Onboard tab:** "What this does: Sets up a new customer..."
- ✅ Each has expandable "How it works" or "Where data is saved"

**Why this matters:** Previous version had minimal/confusing descriptions - users didn't know what tabs were for!

---

## 🧪 **Optional: Test Real Features** (if you have time)

### Optional Test 1: Update Report (5 min)
**Prerequisites:** 
- Real customer with existing tracker
- VPN connected
- JIRA token configured

**Steps:**
1. Go to Update tab
2. Select a customer
3. Click "Update from JIRA"
4. Wait for completion

**Expected:**
- ✅ Shows spinner while processing
- ✅ Success message with file paths
- ✅ Can expand "📁 File locations" to see:
  - Updated: ~/Documents/rh/customer/rfe-bug-tracker.md
  - Backup: ~/Documents/rh/customer/rfe-bug-tracker.md.backup

**NEW in v1.9.5:** File paths are now shown!

---

### Optional Test 2: Vault Token Save (3 min)
**Prerequisites:**
- HashiCorp Vault configured
- VAULT_ADDR and VAULT_TOKEN environment variables set

**Steps:**
1. Go to Vault tab
2. Check connection status
3. Try to add a token (if you want)

**Expected:**
- ✅ Shows Vault connection status
- ✅ Displays VAULT_ADDR from environment

**Known Limitation:** Viewing and deleting tokens not fully implemented yet

---

### Optional Test 3: Settings Persistence (2 min)
**Steps:**
1. Go to Settings tab
2. Change TAM email
3. Check "Auto-update"
4. Click "Save Settings"
5. Close Taminator
6. Reopen Taminator
7. Go to Settings tab

**Expected:**
- ✅ Settings you changed are still there

**FIXED in v1.9.4:** Settings now persist across restarts

---

## ❌ **What NOT to Test** (Known Limitations)

### Don't Waste Time On:
1. ❌ **Dashboard real data** - It's demo data (clearly marked)
2. ❌ **Recent Activity** - It's fake (clearly marked)
3. ❌ **GitHub issue submission** - Not implemented (removed)
4. ❌ **Debug mode checkbox** - Doesn't do anything yet
5. ❌ **Directory browser button** - Not implemented
6. ❌ **View/Delete Vault tokens** - Not fully working

These are **clearly disclosed** in the UI now!

## 🐛 **What to Report if You Find Issues**

### Report These Issues:
1. ✅ **App crashes or freezes**
2. ✅ **Blank screens or missing content**
3. ✅ **Warnings/labels not showing up**
4. ✅ **Buttons that do nothing**
5. ✅ **Confusing or misleading text**

### How to Report:
**Option 1 - Email:**
```

What did you expect?
What actually happened?
```

#tam-automation

## 📊 **Testing Checklist**

### Core Tests (Everyone should do these)
- [ ] App launches successfully
- [ ] Dashboard shows warning banner
- [ ] Dashboard cards labeled "(Demo Data)" or "(Live)"
- [ ] Update tab has NO checkboxes
- [ ] Update tab has blue info box
- [ ] Report Issue tab shows contact methods (not form)
- [ ] All tabs have "What this does" descriptions

### Optional Tests (If you have time/setup)
- [ ] Update report shows file paths
- [ ] Settings persist across restarts
- [ ] Vault shows connection status

---

## ⏱️ **Time Estimates**

- **Quick smoke test:** 5 minutes (just core tests)
- **Thorough test:** 15 minutes (core + optional)
- **Full integration test:** 30 minutes (with real data)

## 💡 **Testing Tips**

1. **First time?** Just do the Core Tests - takes 5 min
2. **Found something?** Screenshot + description helps a lot
3. **Something confusing?** That's a bug too - let us know!
4. **Can't test real features?** That's OK - focus on UI/warnings
5. **App crashes?** Check console for errors (F12 in the app)
