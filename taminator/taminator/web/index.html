When to use debug

If Check or Update shows wrong or missing data, check Debug: include Hydra/JIRA diagnostics and run again. The result will include:

  • Hydra response — SOLR query, numFound, doc count, first doc keys (what the case search API returned).
  • Hydra JIRA — For cases with no JIRA: top-level doc keys and external trackers / attached resources data returned for that case.
  • JIRA API — Per-issue HTTP status and error/body when not 200 (what the JIRA API returned).
  • rhcase — When Update uses rhcase fallback: raw stdout/stderr from rhcase (what rhcase returned).

Check vs Update

Check report — Compares your report to JIRA and shows you any status differences. Does not change the report file.

Update report — Fetches current statuses from JIRA and writes them into the report file (and can create or populate a new report if it’s empty). Creates a backup before changing the file.
